## Plan: Pemisahan Pelanggan & Pengguna (Multi-Role Staff)

### Tujuan
- **Pelanggan** (Customers) = user dengan role `"user"` (guest checkout). Mereka order layanan.
- **Pengguna** (Staff) = user dengan role `"admin"`, `"finance"`, `"support"`. Mereka mengelola sistem.

### Perubahan

#### 1. Schema & Role
Tidak perlu migrasi DB — kolom `role` sudah text. Role baru: `admin`, `finance`, `support`.
- `finance`: hanya bisa lihat transaksi, invoice, settings payment
- `support`: hanya bisa lihat pelanggan, subscription, paket & layanan
- `admin`: full akses

#### 2. Backend Middleware
**`server/src/index.ts`** — ganti validasi role dari `!== "admin"` jadi whitelist multi-role:
```ts
if (user.role !== "admin" && user.role !== "finance" && user.role !== "support") {
  return c.json({ error: "Forbidden" }, 403);
}
```

**`server/src/routes/admin.ts`** — tambah per-route permission:
- `GET /stats`, `GET /transactions`, `PATCH /:id/confirm` → `admin|finance`
- `GET /users`, `GET /users/:id` → `admin|support`
- Services/Packages CRUD → `admin|support`
- `POST /activate-subscription` → `admin`

#### 3. Frontend Admin Layout
**`client/src/components/layout/AdminLayout.tsx`** — pisahkan menu:
- **Ringkasan** (Overview) → semua role
- **Pelanggan** (Customers) → `admin|support` — daftar & detail pelanggan
- **Transaksi** → `admin|finance`
- **Paket & Layanan** → `admin|support`
- **Settings** → `admin`
- **Pengguna** (Staff) → `admin` only — kelola staff (tambah, edit role, hapus)

#### 4. Halaman Baru
- **`/admin/pengguna`** — Hanya untuk admin. Daftar staff (role admin/finance/support). Bisa menambah staff baru dan mengubah role.
- **`/admin/pelanggan`** — Untuk admin & support. Daftar semua pelanggan (role user). Sama seperti `/admin/users` sebelumnya.

Route `/admin/users` diarahkan ke `/admin/pelanggan` untuk clarity.

#### 5. Endpoint Baru
- `GET /api/admin/staff` — daftar staff (role admin/finance/support)
- `POST /api/admin/staff` — tambah staff baru (admin-only)
- `PATCH /api/admin/staff/:id` — edit role staff (admin-only)
