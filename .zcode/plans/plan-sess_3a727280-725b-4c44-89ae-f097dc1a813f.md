## Plan: Guest Checkout (Shadow User) & Integrasi Fonnte WA

Sistem wajib login akan dihapus untuk checkout. Sebagai gantinya, pelanggan cukup memasukkan Nama, Email, dan WhatsApp. Transaksi dan masa garansi tetap tercatat sempurna.

### Bagian 1: Shadow User & Skema DB
**`server/src/db/schema.ts`**
- Tambah kolom `whatsapp` (text) ke tabel `users`.
- Agar transaksi guest bisa dilacak, backend akan otomatis membuat **Shadow User** (ID acak, tanpa password) jika email belum pernah order. Jika email sudah pernah order, transaksi baru dihubungkan ke ID lama.

### Bagian 2: Update Endpoint Transaksi
**`server/src/routes/transactions.ts`**
- Endpoint `POST /checkout` dan `POST /` diperbarui untuk menerima: `name`, `email`, `whatsapp` (sebagai ganti dari kewajiban mengirim `userId`).
- Logika: 
  1. Cek DB: apakah `email` sudah ada?
  2. Jika ya: pakai ID tersebut, update nomor WA-nya.
  3. Jika tidak: Insert ke tabel `users` dengan ID `guest-<uuid>` dan `role = "user"`.
  4. Lanjutkan pembuatan transaksi ke SumoPod dengan ID tersebut.

### Bagian 3: Integrasi Fonnte WA Gateway
**`server/src/lib/fonnte.ts`** (File baru)
- Helper untuk mengirim pesan via API Fonnte (`https://api.fonnte.com/send`).
- Menggunakan API Key dari environment variable `FONNTE_TOKEN`.

**`server/src/routes/webhooks.ts` & `admin.ts`**
- Saat pembayaran **otomatis sukses (QRIS)** atau **dikonfirmasi manual oleh admin**:
  - Kirim WA ke **Admin**: "Pesanan Baru Lunas! Paket: X, Pelanggan: Y, Nominal: Z".
  - Kirim WA ke **Pelanggan**: "Terima kasih! Pembayaran Anda sebesar Rp Z untuk paket X telah berhasil. Masa garansi aktif hingga [Tanggal]. Tim kami akan segera menghubungi Anda."

### Bagian 4: Hapus Wajib Login dari Frontend
**`client/src/pages/services/Malware.tsx` & `Layanan.tsx`**
- Hapus input `password`.
- Hapus logika pendaftaran akun via Neon Auth (`authClient.signUp.email`).
- Langsung tembak endpoint checkout dengan mengirimkan `name, email, whatsapp`.
- Halaman dashboard (`/dashboard`) tetap dilindungi login (hanya untuk user yang memang sengaja daftar dari halaman Register, atau jika kelak admin memberi akses member).

### ⚠️ Persiapan yang Dibutuhkan Setelah Selesai
1. Anda perlu menjalankan `cd server && npm run db:push` untuk menambahkan kolom `whatsapp`.
2. Anda perlu menambahkan token Fonnte di `.env` server: `FONNTE_TOKEN=your-token-here`.
3. Anda perlu menambahkan nomor WA admin di `.env` server: `ADMIN_WHATSAPP=628xxxx`.
